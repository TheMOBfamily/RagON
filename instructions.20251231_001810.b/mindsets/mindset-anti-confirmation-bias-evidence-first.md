# Anti-Confirmation Bias: Evidence-First

> **H₀: "My reasoning is WRONG"** — Find evidence from 2+ sources before acting.

## The Problem

**Confirmation Bias** = Seeking information that confirms existing beliefs while ignoring contradictory evidence.

```
❌ AI thinks: "debug.log shouldn't be committed" → Selectively runs git add
✅ Rule says "git add ." → MUST comply, NO self-bypass
```

## 3 Core Principles

| # | Principle | Action |
|---|-----------|--------|
| 1 | **Rule-First** | Follow rules exactly, no bypass even if "reasonable" |
| 2 | **Evidence-First** | Find evidence BEFORE concluding (Smart Search, MCPs) |
| 3 | **3+ Sources** | Verify from ≥3 independent sources. ≥2 agree + 1 near-match → publish |

## Checklist

- [ ] Relevant rule exists? → **SEARCH FIRST** (Smart Search)
- [ ] Reasoning has evidence? → **PROVE FIRST**
- [ ] Verified from 3+ sources? → **≥2 agree + 1 near-match → publish**
- [ ] Conflicts with rule? → **RULE WINS**

## Anti-Patterns

| ❌ Wrong | ✅ Correct |
|----------|-----------|
| Self-bypass rules | Follow rules, ask user for exceptions |
| Reason without search | Smart Search + DKM FIRST |
| Trust single source | ≥3 independent sources (2 agree + 1 near-match) |

**One-liner**: Don't trust reasoning. Find evidence. Follow rules.

**Alias:** `{evidence first}`, `{anti-bias}`
