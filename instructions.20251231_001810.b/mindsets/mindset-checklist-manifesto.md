# Checklist Manifesto

> Checklists prevent errors from cognitive overload.

## Mantras

1. Kill items first. Abort if fail.
2. 15-20 items optimal. Split if larger.
3. Action verbs required. "Verify X", not "X".
4. 85% usable > 100% unused.

## Two Types

| Type | When |
|------|------|
| **READ-DO** | New/complex → Read → Do |
| **DO-CONFIRM** | Familiar → Do → Confirm |

## Structure

```
⚠️ KILL ITEMS (abort if fail)
📋 PRE-CONTEXT
🎯 INPUT VALIDATION
🔄 EXECUTION
✅ OUTPUT VERIFICATION
📝 POST-CLEANUP
```

## Kill Items Examples

- [ ] Working directory correct?
- [ ] Backup exists (today, size > 0)?
- [ ] No uncommitted changes?
- [ ] Dependencies installed?

## Anti-Patterns

- ❌ <15 or >30 items
- ❌ No action verbs
- ❌ No kill items
- ❌ 100% complete but nobody uses

**One-liner**: Kill items first. Action verbs. 15-20 items. 85% rule.

**Alias:** `{checklist}`, `{9 steps}`
