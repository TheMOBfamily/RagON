# Double-Check & Cross-Check

> Verify TWICE. Verify INDEPENDENTLY. Use DIFFERENT AI.

## Mantras

1. Assume you're WRONG. Verify again.
2. Same method twice = Double-Check.
3. Different method = Cross-Check.
4. Same AI verifying itself = Confirmation Bias.
5. Different AI = Independent verification.

## Definitions

| Method | What | How |
|--------|------|-----|
| Double-Check | Same thing, twice | Repeat process |
| Cross-Check | Different tool | Independent verify |

## Examples

```
Double: Count → Count again → Compare
Cross:  API updates → Query DB directly (not API)
```

## Independent Tools

| Tool | When |
|------|------|
| queryPerplexity | Factual claims |
| copilot-free.sh | Code logic |
| Both | High-stakes decisions |

## Prompt Principles

1. Role: Context-appropriate verifier
2. Stance: Assume WRONG. Seek to disprove.
3. Output: Structured. With sources.
4. Adapt: Generate prompt per context

## Checklist (9 items)

- [ ] Double-checked result?
- [ ] Cross-checked with different tool?
- [ ] Used different AI?
- [ ] Perplexity queried?
- [ ] Copilot queried?
- [ ] Both agree?
- [ ] Sources cited?
- [ ] Confidence noted?
- [ ] Discrepancies investigated?

**One-liner**: Same thing twice. Different AI once. Trust neither until both agree.

**Alias:** `{double check}`, `{verify twice}`
